列表群組 (List group)

列表群組是一個具有彈性的元件，用於顯示一系列內容。可以修改、擴增以支援更多的內容。

六角學院的 Bootstrap 5 課程上線囉，立即與萬人一同學習最專業、最深入的 Bootstrap 5 課程。

基礎範例

最基本的列表群組是具有列表項目和適當 Class 的無序列表。根據以下的選項使用它，或根據您的需要調整 CSS。

  • An item
  • A second item
  • A third item
  • A fourth item
  • And a fifth one
<ul class="list-group">
  <li class="list-group-item">An item</li>
  <li class="list-group-item">A second item</li>
  <li class="list-group-item">A third item</li>
  <li class="list-group-item">A fourth item</li>
  <li class="list-group-item">And a fifth one</li>
</ul>

啟用物件

將 .active 加到 .list-group-item 指示當前的活動狀態。

  • An active item
  • A second item
  • A third item
  • A fourth item
  • And a fifth one
<ul class="list-group">
  <li class="list-group-item active" aria-current="true">An active item</li>
  <li class="list-group-item">A second item</li>
  <li class="list-group-item">A third item</li>
  <li class="list-group-item">A fourth item</li>
  <li class="list-group-item">And a fifth one</li>
</ul>

禁用選項

將 .disabled 加到 .list-group-item 使其 呈現 禁用。請注意，一些帶有 .disabled 的元素還需要自訂 JavaScript 才能完全禁用其點擊事件（例如：連結）。

  • A disabled item
  • A second item
  • A third item
  • A fourth item
  • And a fifth one
<ul class="list-group">
  <li class="list-group-item disabled" aria-disabled="true">A disabled item</li>
  <li class="list-group-item">A second item</li>
  <li class="list-group-item">A third item</li>
  <li class="list-group-item">A fourth item</li>
  <li class="list-group-item">And a fifth one</li>
</ul>

連結和按鈕

使用 <a> 或 <button> ，並加入 .list-group-item-action 來創建具有 hover、禁用和啟用狀態的動態列表群組。我們分離這些偽類別，以確保由非交互元素組成的列表群組（如 <li> 或 <div> ）不提供點擊或觸擊。

確保不要在這裡使用標準的 .btn。

The current link item A second link item A third link item A fourth link item A disabled link item 
<div class="list-group">
  <a href="#" class="list-group-item list-group-item-action active" aria-current="true">
    The current link item
  </a>
  <a href="#" class="list-group-item list-group-item-action">A second link item</a>
  <a href="#" class="list-group-item list-group-item-action">A third link item</a>
  <a href="#" class="list-group-item list-group-item-action">A fourth link item</a>
  <a class="list-group-item list-group-item-action disabled">A disabled link item</a>
</div>

對於 <button>，你也可以使用 disabled 屬性而不是 .disabled class。 不幸的是，<a> 不支援 disabled 屬性。

<div class="list-group">
  <button type="button" class="list-group-item list-group-item-action active" aria-current="true">
    The current button
  </button>
  <button type="button" class="list-group-item list-group-item-action">A second item</button>
  <button type="button" class="list-group-item list-group-item-action">A third button item</button>
  <button type="button" class="list-group-item list-group-item-action">A fourth button item</button>
  <button type="button" class="list-group-item list-group-item-action" disabled>A disabled button item</button>
</div>

Flush

加入 .list-group-flush 來移除部分邊框及圓角，以在父容器中 (如：卡片) 產生邊緣貼齊的列表群組。

  • An item
  • A second item
  • A third item
  • A fourth item
  • And a fifth one
<ul class="list-group list-group-flush">
  <li class="list-group-item">An item</li>
  <li class="list-group-item">A second item</li>
  <li class="list-group-item">A third item</li>
  <li class="list-group-item">A fourth item</li>
  <li class="list-group-item">And a fifth one</li>
</ul>

編號

添加 .list-group-numbered 類別 (並可選擇使用 <ol> 元素) 以選擇加入編號列表群組。數字是通過 CSS 來生成的 (而不是 <ol> 的預設瀏覽器樣式) ，以便更好的放置在列表群組中以及自定義。

數字由 <ol> 上的 counter-reset 生成，然後使用 ::before 偽元素設置樣式並放在 <li> 上，帶有 counter-increment 和 content。

  1. A list item
  2. A list item
  3. A list item
<ol class="list-group list-group-numbered">
  <li class="list-group-item">A list item</li>
  <li class="list-group-item">A list item</li>
  <li class="list-group-item">A list item</li>
</ol>

Conveying meaning to assistive technologies

Using color to add meaning only provides a visual indication, which will not be conveyed to users of assistive technologies – such as screen readers. Ensure that information denoted by the color is either obvious from the content itself (e.g. the visible text), or is included through alternative means, such as additional text hidden with the .visually-hidden class.

Mixins

與 $theme-colors 結合使用來生成 .list-group-item 的 contextual variant classes。

@mixin list-group-item-variant($state, $background, $color) {
  .list-group-item-#{$state} {
    color: $color;
    background-color: $background;

    &.list-group-item-action {
      &:hover,
      &:focus {
        color: $color;
        background-color: shade-color($background, 10%);
      }

      &.active {
        color: $white;
        background-color: $color;
        border-color: $color;
      }
    }
  }
}

Loop

使用 list-group-item-variant() mixin 生成修飾符類別的 loop。

// List group contextual variants
//
// Add modifier classes to change text and background color on individual items.
// Organizationally, this must come after the `:hover` states.

@each $state, $value in $theme-colors {
  $list-group-variant-bg: shift-color($value, $list-group-item-bg-scale);
  $list-group-variant-color: shift-color($value, $list-group-item-color-scale);
  @if (contrast-ratio($list-group-variant-bg, $list-group-variant-color) < $min-contrast-ratio) {
    $list-group-variant-color: mix($value, color-contrast($list-group-variant-bg), abs($list-group-item-color-scale));
  }

  @include list-group-item-variant($state, $list-group-variant-bg, $list-group-variant-color);
}

透過 JavaScript

通過 JavaScript 啟用可選的列表項目（每個列表項目需要單獨啟動）：

var triggerTabList = [].slice.call(document.querySelectorAll('#myTab a'))
triggerTabList.forEach(function (triggerEl) {
  var tabTrigger = new bootstrap.Tab(triggerEl)

  triggerEl.addEventListener('click', function (event) {
    event.preventDefault()
    tabTrigger.show()
  })
})

您可以透過以下幾種方式啟動個別列表項目：

var triggerEl = document.querySelector('#myTab a[href="#profile"]')
bootstrap.Tab.getInstance(triggerEl).show() // Select tab by name

var triggerFirstTabEl = document.querySelector('#myTab li:first-child a')
bootstrap.Tab.getInstance(triggerFirstTabEl).show() // Select first tab

方法

constructor

啟動列表項目元素和內容容器。頁籤應該有一個 data-bs-target 或 href 定位在 DOM 中的一個容器節點。

  <a class="list-group-item list-group-item-action" data-bs-toggle="list" href="#settings" role="tab">Settings</a>
</div>

<div class="tab-content">
  <div class="tab-pane active" id="home" role="tabpanel">...</div>
  <div class="tab-pane" id="profile" role="tabpanel">...</div>
  <div class="tab-pane" id="messages" role="tabpanel">...</div>

<script>
  var firstTabEl = document.querySelector('#myTab a:last-child')
  var firstTab = new bootstrap.Tab(firstTabEl)

  firstTab.show()
</script>

show

選擇給定的列表項目並顯示其關聯的 pane。而先前選擇的任何其他列表項目將被取消，並使其關聯的 pane 隱藏。在 Tab pane 顯示之前回傳給調用者（例如，在 shown.bs.tab 事件發生之前）。

  var someListItemEl = document.querySelector('#someListItem')
  var tab = new bootstrap.Tab(someListItemEl)

  tab.show()

dispose

銷毀元素的頁籤。

getInstance

靜態 方法可以使您獲取與 DOM 元素相關聯的實例。

var triggerEl = document.querySelector('#trigger')
var tab = bootstrap.Tab.getInstance(triggerEl) // Returns a Bootstrap tab instance

getOrCreateInstance

靜態 方法，允許你獲取與 DOM 元素相關的 tab 實例，或者在未初始化的情況下創建一個新實例。

var triggerEl = document.querySelector('#trigger')
var tab = bootstrap.Tab.getOrCreateInstance(triggerEl) // Returns a Bootstrap tab instance

事件

當顯示新分頁時，事件按以下順序觸發：

  1. hide.bs.tab (在當前啟用的頁籤上)
  2. show.bs.tab (在準備啟用的頁籤上)
  3. hidden.bs.tab (在先前啟用的頁籤上，與 hide.bs.tab 事件相同)
  4. shown.bs.tab (在剛剛顯示的頁籤上，與 show.bs.tab 事件相同)

如果沒有選項啟用，則 hide.bs.tab 和 hidden.bs.tab 事件將不被觸發。

Event type Description
show.bs.tab 這個事件因頁籤顯示而觸發，但是會在新頁籤被顯示之前結束。 使用 event.target 和 event.relatedTarget 將目前與先前啟用的頁籤(如果可用) 的作為目標。
shown.bs.tab 這個事件在頁籤顯示後觸發。 使用 event.target 和 event.relatedTarget 來分別定位啟用中和剛剛啟用的頁籤。
hide.bs.tab 新的頁籤要顯示（先前的啟用頁籤將被隱藏）時，此事將觸發。 使用 event.target 和 event.relatedTarget 分別定位先前啟用和即將啟用的頁籤。
hidden.bs.tab 新的選項顯示後（因此先前啟用的選項將隱藏），此事件將觸發。 使用 event.target 和 event.relatedTarget 分別定位先前啟用的頁籤和新啟用的頁籤 。 
var tabElms = document.querySelectorAll('a[data-bs-toggle="list"]')
tabElms.forEach(function(tabElm) {
  tabElm.addEventListener('shown.bs.tab', function (event) {
    event.target // newly activated tab
    event.relatedTarget // previous active tab
  })
}
